### Fixed
- `verify_runtime_release.py`: notarization is now verified with the code-object-native `codesign --verify --strict --test-requirement '=notarized'` instead of `spctl --assess --type execute`. The runtime entrypoint is a bare command-line Mach-O, not a `.app` bundle, and `spctl --type execute` only assesses `.app` bundles — it always rejects a bare executable ("the code is valid but does not seem to be an app"), a false negative that would fail every otherwise-valid activation release at the notarization step. The `notarized` codesign requirement binds to the CDHash and still rejects unsigned, ad-hoc, and Developer-ID-signed-but-not-notarized binaries by exit code.
