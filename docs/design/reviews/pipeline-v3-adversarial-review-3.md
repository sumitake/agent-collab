GitHub attestations can provide signed provenance and allow consumers to pin the signer workflow; public-repository attestations are also written to an immutable transparency log. A run ID plus matching public input is insufficient. [GitHub attestation verification](https://docs.github.com/en/actions/how-tos/secure-your-work/use-artifact-attestations/increase-security-rating), [attestation properties](https://docs.github.com/en/enterprise-cloud@latest/actions/concepts/security/artifact-attestations)

The immutable tag preserves the recovery root but not the rollback decision. Commit the signed burn or a signed `rollback-started` record to protected `main` before the first destructive compensation. Once that record is durable, cleanup can be safely re-driven.
